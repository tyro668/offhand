# 首次使用向导设计

## 目标

首次启动 Offhand 时，用户需要在一个连续流程里完成三件事：

1. 理解并设置启动听写的快捷键。
2. 配置至少一个语音识别大模型。
3. 配置文本增强大模型，并决定是否启用文本增强。

向导不是额外的一套配置系统，而是现有设置的入口聚合。向导中保存的结果必须直接写入当前应用已有配置，完成后主界面和设置页立即反映同一份状态。

## 触发规则

- 新增 `onboarding_completed` 持久化标记。
- `SettingsProvider.load()` 完成后，如果标记不存在或为 `false`，主界面自动弹出首次向导。
- 向导不可通过点击遮罩误关闭，用户需要点击“稍后设置”或“完成向导”。
- 用户选择“稍后设置”也会写入完成标记，避免每次启动都打扰；后续仍可在设置页继续配置。
- 通用设置提供“下次启动时显示启动向导”测试项。开启后只影响下一次应用启动，不在当前设置会话中立即弹出。

## 信息架构

向导采用 3 步：

### 1. 启动快捷键

本步骤提供：

- 当前快捷键展示。
- 点击后捕获新按键。
- 恢复默认快捷键。
- 激活模式选择。

保存行为复用：

- `SettingsProvider.setHotkey`
- `SettingsProvider.resetHotkey`
- `SettingsProvider.setActivationMode`

### 2. 语音识别大模型

本步骤提供：

- 当前已启用语音模型状态。
- 服务商选择。
- 模型选择。
- 云端模型 API Key 输入。
- 本地模型选择入口；本地模型下载仍保留在语音模型页中，避免首次向导承担大文件下载生命周期。

保存行为：

- 创建 `SttModelEntry`。
- 调用 `SettingsProvider.addSttModelEntry`。
- 调用 `SettingsProvider.enableSttModelEntry` 确保新模型成为当前模型。

### 3. 文本增强大模型

本步骤提供：

- 当前已启用文本模型状态。
- 是否启用文本增强开关。
- 服务商选择。
- 模型选择。
- API Key 输入。
- 自定义 OpenAI 兼容端点。

保存行为：

- 创建 `AiModelEntry`。
- 调用 `SettingsProvider.addAiModelEntry`。
- 调用 `SettingsProvider.enableAiModelEntry`。
- 调用 `SettingsProvider.setAiEnhanceEnabled`。

## 完成条件

- 用户可在任意时刻选择“稍后设置”退出，不阻塞进入应用。
- “下一步”不强制要求模型配置完成，但页面会展示当前配置状态。
- “保存语音模型”和“保存文本模型”只有在必要字段完整时可点击。
- “完成向导”写入完成标记并关闭弹窗。

## UI 原则

- 首屏直接呈现实际配置控件，不做欢迎页或功能介绍。
- 页面只保留标题、状态和必要表单字段。
- 步骤标题使用短词，不显示长段描述。
- 视觉保持留白、清晰边界和轻量控件，避免大面积说明文字。

## 非目标

- 不在首次向导中下载本地 ASR 大模型；本地模型下载、进度和文件管理继续由语音模型设置页负责。
- 不在首次向导中测试模型连接；连接测试继续由模型设置页负责。
- 不引入新的模型配置存储格式。

## 验收点

1. 新安装或无完成标记时，应用加载完成后自动显示首次向导。
2. 设置快捷键后，通用设置页显示同一快捷键。
3. 保存语音模型后，语音模型页显示该模型并处于启用状态。
4. 保存文本模型后，文本模型页显示该模型并处于启用状态。
5. 启用文本增强后，录音完成流程使用当前文本增强配置。
6. 点击“稍后设置”或“完成向导”后，下次启动不再自动弹出。
7. 在通用设置中开启“下次启动时显示启动向导”后，当前会话不弹出向导，下次启动会重新显示。
